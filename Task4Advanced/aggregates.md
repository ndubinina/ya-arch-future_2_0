# Агрегаты доменной модели

## Доммен медицинских услуг

### Агрегат «Пациент»
* **Описание:** Управление данными пациента. Хранит персональные и медицинские данные.
* **Бизнес-инварианты (Правила целостности):**
    * У пациента не может быть двух действующих медицинских карт в системе.
    * Номер телефона должен быть уникальным для исключения дубликатов.
    * Изменение персональных данных фиксируется в аудите
* **Идентификаторы и Ключи:**
    * `patient_id` — Первичный ключ (UUID).
    * `phone_number` — Уникальный ключ.


### Агрегат: Визит (Visit)
* **Описание:** Управление процессом обращения пациента в клинику. Хранит статус и состав визита.
* **Бизнес-инварианты (Правила целостности):**
    * Визит не может быть запланирован на прошедшее время.
    * Визит может находиться только в одном статусе (назначен, завершён, отменён).
* **Идентификаторы и Ключи:**
    * `visit_id` — Первичный ключ (UUID).
    * `patient_id` — Внешний ключ.


## Домен ИИ анализа

### Агрегат: ИИ-Исследование
* **Описание:** Процесс обработки медицинских данных пациента с помощью ML-моделей и генерации даигностики.
* **Бизнес-инварианты (Правила целостности):**
    * Результат диагностики связан с конкретной версией модели.
    * Для одной диагностической сессии может существовать только один итоговый результат.
* **Идентификаторы и Ключи:**
    * `diagnostic_id` — Идентификатор исследования (UUID).
    * `patient_id` — Внешний ключ.
    * `ai_model_id` - Внешний ключ.


## Финтех домен

### Агрегат: Кредит 
* **Описание:** Управление жизненным циклом кредита клиента.
* **Бизнес-инварианты (Правила целостности):**
    * Сумма кредита не может превышать установленный лимит.
    * Закрытый кредит не может быть повторно активирован.
* **Идентификаторы и Ключи:**
    * `loan_id` — Идентификатор договора (UUID) .
    * `patient_id` — Внешний ключ.
